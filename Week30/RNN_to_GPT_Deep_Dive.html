I

Recurrent Neural Networks

Understanding the foundation of sequential deep learning — where memory meets computation

The Problem

Why Standard Neural Networks Fail at Sequences

Traditional feedforward neural networks assume that all inputs are independent of each other. But real-world data is deeply sequential:

  • Language: "The cat sat on the ___" — the next word depends on all previous words
  • Time Series: Stock prices depend on historical trends and momentum
  • Speech: Phonemes only make sense in the context of surrounding sounds
  • Music: Each note depends on the melody, rhythm, and harmony that preceded it

Feedforward networks have no memory — they process each input in isolation. We need an architecture that can maintain an internal state across time steps.

Core Limitations of FFNNs

  • Fixed input size — cannot handle variable-length sequences
  • No parameter sharing — learns separate weights for each position
  • No temporal dynamics — ignores ordering of inputs entirely
  • Combinatorial explosion — representing all possible orderings is intractable
Feedforward: Memoryless y = f(x) — output depends ONLY on current input
Core Idea

The Recurrent Neural Network

An RNN introduces a recurrence relation — the output at each time step depends not only on the current input, but also on the hidden state from the previous time step.

This hidden state acts as the network's memory, carrying a compressed summary of all inputs seen so far.

The Recurrence — The Heart of RNN hₜ = f(hₜ₋₁, xₜ)

This single equation is revolutionary: the same function f is applied at every time step, with shared weights. This gives RNNs three superpowers:

  • Handle variable-length sequences
  • Share parameters across time (fewer weights)
  • Maintain temporal context

Key Insight — Weight Sharing

The same three weight matrices are used at every single time step. This is what makes RNNs fundamentally different from feedforward networks — the number of parameters does not grow with sequence length.

Three Weight Matrices

  • Wₓₕ — input-to-hidden weights
  • Wₕₕ — hidden-to-hidden (recurrent) weights
  • Wₕᵧ — hidden-to-output weights
Mathematics

RNN Forward Pass — Full Equations

At each time step t, the RNN computes two things: the new hidden state and the output.

Step 1 — Hidden State Update hₜ = tanh(Wₓₕ · xₜ + Wₕₕ · hₜ₋₁ + bₕ)

This combines the current input xₜ with the previous memory hₜ₋₁ through a non-linear activation. The tanh squashes values to the range [-1, +1], enabling the network to represent both positive and negative features.

Step 2 — Output Computation yₜ = softmax(Wₕᵧ · hₜ + bᵧ)

The output transforms the hidden state into a probability distribution over the output vocabulary (for classification) or a continuous value (for regression).

Dimensions

SymbolShapeMeaning
xₜd × 1Input vector at time t
hₜn × 1Hidden state at time t
yₜk × 1Output at time t
Wₓₕn × dInput weight matrix
Wₕₕn × nRecurrent weight matrix
Wₕᵧk × nOutput weight matrix

where d = input dimension, n = hidden units, k = output classes

Deep Dive

The Hidden State — RNN's Memory

The hidden state hₜ is the most important concept in RNNs. Let us expand it recursively to understand what it truly encodes:

Recursive Expansion h₁ = tanh(Wₓₕ·x₁ + Wₕₕ·h₀ + b)
h₂ = tanh(Wₓₕ·x₂ + Wₕₕ·h₁ + b)
= tanh(Wₓₕ·x₂ + Wₕₕ·tanh(Wₓₕ·x₁ + Wₕₕ·h₀ + b) + b)
h₃ = tanh(Wₓₕ·x₃ + Wₕₕ·h₂ + b)
= tanh(Wₓₕ·x₃ + Wₕₕ·tanh(Wₓₕ·x₂ + Wₕₕ·tanh(...) + b) + b)

Each hidden state is a deeply nested composition of all previous inputs. The information from x₁ must survive through multiple layers of tanh and matrix multiplication to influence hₜ.

Critical Insight

The hidden state is a lossy compression of the entire input history. It is a fixed-size vector (n dimensions) that must somehow encode an arbitrary-length sequence.

This is both the power and the fundamental limitation of RNNs — the information bottleneck means that information from early time steps gets progressively overwritten.

Why tanh?

  • Zero-centered: outputs in [-1, +1], enabling both positive and negative activations
  • Bounded: prevents hidden states from exploding to infinity
  • Differentiable: required for gradient-based learning
  • Derivative: tanh′(x) = 1 − tanh²(x), maximum of 1 at x=0
Training

RNN Loss Function

The total loss across a sequence of length T is the sum of losses at each individual time step:

Total Sequence Loss L = Σₜ₌₁ᵀ Lₜ(yₜ, ŷₜ)

For classification tasks using cross-entropy:

Cross-Entropy at Time t Lₜ = −Σⱼ ŷₜⱼ · log(yₜⱼ)

The goal of training is to find weight matrices Wₓₕ, Wₕₕ, Wₕᵧ that minimize this total loss across all time steps and all training sequences.

RNN Output Modes

  • Many-to-Many: output at every time step (e.g., POS tagging — compute loss at each t)
  • Many-to-One: output only at the final step (e.g., sentiment analysis — loss only at T)
  • One-to-Many: single input, sequence output (e.g., image captioning)
  • Seq-to-Seq: encoder–decoder, combining many-to-one with one-to-many
Training Objective θ* = argmin_θ Σᵢ₌₁ᴺ L⁽ⁱ⁾(θ)
where θ = {Wₓₕ, Wₕₕ, Wₕᵧ, bₕ, bᵧ}
Core Algorithm

Backpropagation Through Time (BPTT)

BPTT is the algorithm used to compute gradients in RNNs. It is standard backpropagation applied to the unfolded computational graph.

Gradient of Loss w.r.t. Wₕₕ ∂L/∂Wₕₕ = Σₜ₌₁ᵀ ∂Lₜ/∂Wₕₕ

Each ∂Lₜ/∂Wₕₕ requires the chain rule applied backwards through all preceding time steps:

Chain Rule Expansion ∂Lₜ/∂Wₕₕ = Σₖ₌₁ᵗ (∂Lₜ/∂yₜ) · (∂yₜ/∂hₜ) · (∂hₜ/∂hₖ) · (∂hₖ/∂Wₕₕ)

The critical term is ∂hₜ/∂hₖ — the gradient flowing from time t all the way back to time k.

The Dangerous Product

Gradient Through Time ∂hₜ/∂hₖ = Πⱼ₌ₖ₊₁ᵗ ∂hⱼ/∂hⱼ₋₁

where each factor:
∂hⱼ/∂hⱼ₋₁ = diag(tanh′(zⱼ)) · Wₕₕ

This is a product of (t−k) matrices. When this product is computed over many time steps, disaster strikes — leading us to the vanishing gradient problem.

Critical Problem

The Vanishing Gradient Problem

Consider the gradient product term from BPTT:

∂hₜ/∂hₖ = Πⱼ₌ₖ₊₁ᵗ diag(tanh′(zⱼ)) · Wₕₕ

Since tanh′(x) ∈ (0, 1] and is maximum only at x = 0, the diagonal matrix has entries strictly less than 1 for most values.

If the largest eigenvalue of Wₕₕ is less than 1, then:

Exponential Decay ‖∂hₜ/∂hₖ‖ ≤ (γ)ᵗ⁻ᵏ where γ < 1

As (t − k) → ∞, gradient → 0 exponentially

The gradient signal from distant past time steps vanishes to zero, making it impossible for the network to learn long-range dependencies.

Practical Impact

In practice, vanilla RNNs can only learn dependencies spanning roughly 5–10 time steps. Beyond that, gradients are effectively zero.

Example: In the sentence "The cat, which had been sitting on the warm windowsill all morning watching birds, finally stretched." — an RNN cannot learn that "stretched" depends on "cat" because they are too far apart.

Exploding Gradient (the flip side)

If the largest eigenvalue of Wₕₕ is greater than 1, gradients explode exponentially. This is easier to handle — we simply clip gradients:

Gradient Clipping if ‖g‖ > threshold:
g ← (threshold / ‖g‖) · g
Mathematical Analysis

Eigenvalue Analysis of Gradient Flow

The fate of gradients in an RNN is governed by the spectral properties of the recurrent weight matrix Wₕₕ.

Let λ₁, λ₂, ..., λₙ be the eigenvalues of Wₕₕ, with λₘₐₓ being the largest eigenvalue by absolute value (the spectral radius).

Spectral Radius ρ(Wₕₕ) = max|λᵢ|
Three Regimes ρ(Wₕₕ) < 1 → Vanishing gradients (forgetting)
ρ(Wₕₕ) = 1 → Stable gradients (ideal but fragile)
ρ(Wₕₕ) > 1 → Exploding gradients (instability)

Bengio et al. (1994) — Formal Proof

In their seminal paper, Bengio, Simard, and Frasconi proved that for any RNN with bounded activation functions, the gradient of the loss with respect to early hidden states decays exponentially with the number of time steps. Specifically:

‖∂Lₜ/∂hₖ‖ ≤ ηᵗ⁻ᵏ

where η = ‖Wₕₕ‖ · sup|φ′| < 1
for most practical configurations

This is not just a numerical nuisance — it is a fundamental impossibility for standard RNNs to capture long-range dependencies through gradient-based learning.

II

Long Short-Term Memory

Hochreiter & Schmidhuber's 1997 breakthrough — engineering a memory cell that can learn what to remember and what to forget

Motivation

Why LSTM? The Design Philosophy

The vanishing gradient problem tells us that information flowing through repeated multiplication by Wₕₕ and tanh cannot survive over long distances.

The LSTM solution is elegant: create a separate cell state Cₜ that acts as a conveyor belt of information. Instead of multiplying at every step, information flows via addition.

The Key Innovation RNN: hₜ = tanh(W·[hₜ₋₁, xₜ]) ← multiplicative
LSTM: Cₜ = fₜ ⊙ Cₜ₋₁ + iₜ ⊙ C̃ₜ ← additive!

The LSTM uses three gates — differentiable mechanisms that learn to control what information to store, update, and output.

Three Gates, One Cell State

f
Forget Gate — decides what to remove from cell state
i
Input Gate — decides what new information to store
o
Output Gate — decides what to output from cell state
C
Cell State — the long-term memory highway

All gates use the sigmoid function σ, outputting values in [0,1] — a "soft switch" between 0 (block) and 1 (pass through).

Gate 1 of 3

The Forget Gate

The forget gate examines the previous hidden state hₜ₋₁ and current input xₜ and decides, for each element of the cell state, how much to retain.

Forget Gate Equation fₜ = σ(W_f · [hₜ₋₁, xₜ] + b_f)

The output fₜ is a vector of values between 0 and 1:

  • fₜ = 0 → completely forget this cell state dimension
  • fₜ = 1 → completely retain this cell state dimension
  • fₜ = 0.7 → retain 70% of this dimension's value

Notation: [hₜ₋₁, xₜ] denotes concatenation of the two vectors. If hₜ₋₁ ∈ ℝⁿ and xₜ ∈ ℝᵈ, then the concatenation is in ℝⁿ⁺ᵈ.

Intuitive Example

Consider a language model tracking the subject of a sentence. When it encounters a new subject ("The dog ... then the bird"), the forget gate for the "subject" dimensions should output values near 0 — erasing the old subject ("dog") to make room for the new one ("bird").

Meanwhile, dimensions tracking other features (like tense or mood) might have forget gate values near 1, retaining that information.

Dimensions of W_f

W_f ∈ ℝⁿˣ⁽ⁿ⁺ᵈ⁾
b_f ∈ ℝⁿ
fₜ ∈ ℝⁿ (same size as cell state)
Gate 2 of 3

The Input Gate & Candidate Cell State

The input gate has two parts: deciding what to update and creating new candidate values.

Input Gate — What to Update iₜ = σ(W_i · [hₜ₋₁, xₜ] + b_i)
Candidate Cell State — New Values C̃ₜ = tanh(W_C · [hₜ₋₁, xₜ] + b_C)

The input gate iₜ (sigmoid, values in [0,1]) controls which dimensions to update. The candidate C̃ₜ (tanh, values in [-1,1]) contains the proposed new values.

Together, iₜ ⊙ C̃ₜ creates a scaled version of the new information, where ⊙ denotes element-wise (Hadamard) product.

Why Two Separate Components?

This two-step design provides independent control over:

  • What to write — the candidate C̃ₜ proposes new values using tanh (can be positive or negative)
  • How much to write — the input gate iₜ scales each dimension independently via sigmoid

This separation allows the LSTM to add a tiny adjustment to one dimension while making a large update to another — fine-grained memory control.

Dimensions W_i ∈ ℝⁿˣ⁽ⁿ⁺ᵈ⁾, b_i ∈ ℝⁿ, iₜ ∈ ℝⁿ
W_C ∈ ℝⁿˣ⁽ⁿ⁺ᵈ⁾, b_C ∈ ℝⁿ, C̃ₜ ∈ ℝⁿ
The Heart of LSTM

Cell State Update — The Memory Highway

Now we combine the forget gate and input gate to update the cell state. This is the single most important equation in the LSTM:

Cell State Update (The Core Equation) Cₜ = fₜ ⊙ Cₜ₋₁ + iₜ ⊙ C̃ₜ

fₜ ⊙ Cₜ₋₁ — Selective Forgetting

Element-wise multiply old cell state by forget gate. Each dimension is independently scaled between 0 (completely erased) and 1 (perfectly preserved). This is how the LSTM discards outdated information.

iₜ ⊙ C̃ₜ — Selective Writing

Element-wise multiply candidate values by input gate. Only selected dimensions receive new information. This is how the LSTM stores new relevant information.

Why This Solves the Vanishing Gradient

The gradient through the cell state is: ∂Cₜ/∂Cₜ₋₁ = fₜ (a diagonal matrix). When the forget gate is near 1, the gradient flows through undiminished — no matrix multiplication by Wₕₕ, no tanh squashing. The cell state acts as a gradient highway, allowing error signals to travel hundreds of time steps without vanishing. This additive structure is directly analogous to ResNet skip connections.

Gate 3 of 3

The Output Gate

The output gate controls how much of the cell state is exposed as the hidden state output hₜ.

Output Gate oₜ = σ(W_o · [hₜ₋₁, xₜ] + b_o)
Hidden State Output hₜ = oₜ ⊙ tanh(Cₜ)

The cell state Cₜ is first passed through tanh (to normalize values to [-1,1]), then filtered by the output gate. This means the LSTM can store information in its cell state without outputting it — true long-term memory that is not always visible.

Complete LSTM — All Four Equations

The Full LSTM Cell fₜ = σ(W_f · [hₜ₋₁, xₜ] + b_f) ← Forget
iₜ = σ(W_i · [hₜ₋₁, xₜ] + b_i) ← Input
C̃ₜ = tanh(W_C · [hₜ₋₁, xₜ] + b_C) ← Candidate
Cₜ = fₜ ⊙ Cₜ₋₁ + iₜ ⊙ C̃ₜ ← Cell Update
oₜ = σ(W_o · [hₜ₋₁, xₜ] + b_o) ← Output
hₜ = oₜ ⊙ tanh(Cₜ) ← Hidden State
Practical Considerations

LSTM Parameters & Computational Cost

An LSTM has four sets of weight matrices (one per gate + candidate), each of size n × (n + d), plus biases:

Total Parameters Parameters = 4 × [n × (n + d) + n]
= 4n² + 4nd + 4n
= 4n(n + d + 1)

Compared to a vanilla RNN with n² + nd + n parameters for the recurrent part, an LSTM has roughly 4× more parameters. This is the price of gated memory control.

Example: n=512, d=300 RNN: 512(512+300+1) = 416,256
LSTM: 4×512(512+300+1) = 1,665,024

LSTM Gradient Flow Analysis

The gradient of the loss w.r.t. cell state at time k:

∂Cₜ/∂Cₖ = Πⱼ₌ₖ₊₁ᵗ fⱼ

When forget gates are near 1, this product stays close to 1 even for large (t−k). When the network learns that information should be forgotten, forget gates drop toward 0. This gives the LSTM learned control over its own gradient flow — something vanilla RNNs cannot do.

Peephole Connections

A variant (Gers & Schmidhuber, 2000) adds cell state to gate inputs: fₜ = σ(W_f·[Cₜ₋₁, hₜ₋₁, xₜ] + b_f). This lets gates "peek" at the cell state for finer control.

III

Gated Recurrent Unit

Cho et al. (2014) — A streamlined alternative that achieves similar performance with fewer parameters

Architecture

GRU — Elegant Simplification

The GRU merges the cell state and hidden state into a single state hₜ, and reduces three gates to two:

z
Update Gate zₜ — combines forget + input gate functionality. Decides how much past to keep vs how much new to add.
r
Reset Gate rₜ — controls how much past state to expose when computing the candidate. Can "reset" to act like a fresh network.

By merging cell state and hidden state, the GRU has ~25% fewer parameters than LSTM while maintaining comparable performance on most tasks.

Key Design Difference

The LSTM has separate mechanisms for forgetting old info and adding new info (fₜ and iₜ). The GRU couples them:

Coupled Gates Keep = zₜ
Update = (1 − zₜ)

If you keep 70%, you update 30%.
They MUST sum to 1.

In LSTM, fₜ and iₜ are independent — you could keep 90% AND add 80%, causing the cell state to grow unboundedly. GRU's coupling enforces a natural conservation law.

Mathematics

GRU — Complete Mathematical Formulation

Step 1 — Update Gate zₜ = σ(W_z · [hₜ₋₁, xₜ] + b_z)
Step 2 — Reset Gate rₜ = σ(W_r · [hₜ₋₁, xₜ] + b_r)
Step 3 — Candidate Hidden State h̃ₜ = tanh(W · [rₜ ⊙ hₜ₋₁, xₜ] + b)
Step 4 — Final Hidden State hₜ = zₜ ⊙ hₜ₋₁ + (1 − zₜ) ⊙ h̃ₜ

Understanding Each Step

Update gate zₜ: High values mean "keep the old hidden state" (remember). Low values mean "use the new candidate" (update).

Reset gate rₜ: When near 0, it zeroes out the previous hidden state in the candidate computation, forcing the network to ignore past context. When near 1, the full past state is available.

Candidate h̃ₜ: A proposed new state, computed with the reset-filtered past. The reset gate allows the GRU to "start fresh" when it encounters new, unrelated content.

Final interpolation: A smooth blend between old state and candidate, controlled by zₜ. This is a linear interpolation, which provides clean gradient flow.

GRU Parameter Count 3 × [n × (n + d) + n] = 3n(n + d + 1)
~75% of LSTM parameters
Comparison

LSTM vs GRU — Head to Head

Aspect LSTM GRU
Year Introduced 1997 (Hochreiter & Schmidhuber) 2014 (Cho et al.)

Gates Three: Forget, Input, Output Two: Update, Reset

Advanced Architecture

Bidirectional RNNs / LSTMs

Standard RNNs process sequences left-to-right, meaning each hidden state only encodes past context. But for many tasks, future context matters too.

Example: "He said the bank of the ___ was eroding." — is it a river bank or a financial bank? The word after the blank determines the meaning.

Bidirectional Hidden States h⃗ₜ = f(W⃗ · [h⃗ₜ₋₁, xₜ]) ← Forward
h⃖ₜ = f(W⃖ · [h⃖ₜ₊₁, xₜ]) ← Backward

hₜ = [h⃗ₜ ; h⃖ₜ] ← Concatenation

The final hidden state at each position captures both past and future context, effectively doubling the representational power.

Forward:

h⃗₃
→
h⃗₄
Backward:

←
h⃖₂
←
h⃖₃
←

Trade-offs

  • Advantage: Full context at every position
  • Advantage: Significantly better for classification, NER, POS tagging
  • Limitation: Cannot be used for autoregressive generation (future tokens are unavailable)
  • Limitation: 2× parameters and compute of unidirectional
Application Architecture

Sequence-to-Sequence (Encoder–Decoder)

Sutskever et al. (2014) introduced the Seq2Seq framework for mapping an input sequence to an output sequence of different length. This unlocked machine translation, summarization, and dialogue.

The Bottleneck Problem

The entire input sequence is compressed into a single fixed-size vector c = hₜ (the encoder's final hidden state). For long sequences, this is a severe information bottleneck.

c = encoder(x₁, x₂, ..., xₜ) ∈ ℝⁿ
y₁, y₂, ..., yₛ = decoder(c)

A 50-word English sentence and a 200-word paragraph both get squeezed into the same n-dimensional vector. Performance degrades sharply for long inputs.

This bottleneck directly motivated the invention of the attention mechanism.

IV

The Attention Revolution

Bahdanau et al. (2015) — "Don't compress everything into one vector. Let the decoder look back at the entire input."

Breakthrough

The Attention Mechanism

Instead of forcing the decoder to rely on a single context vector, attention allows the decoder to dynamically focus on different parts of the input at each output step.

Attention Score (Alignment) eₜⱼ = score(sₜ₋₁, hⱼ)
Attention Weights (Softmax) αₜⱼ = exp(eₜⱼ) / Σₖ exp(eₜₖ)
Context Vector (Weighted Sum) cₜ = Σⱼ αₜⱼ · hⱼ

The context vector cₜ is a different weighted combination of encoder states for each decoder step — the network learns to "attend" to relevant input positions.

Score Functions

Common Scoring Methods Dot: score(s, h) = sᵀh
General: score(s, h) = sᵀWₐh
Additive: score(s, h) = vₐᵀ tanh(Wₐ[s;h])
Scaled Dot: score(s, h) = sᵀh / √dₖ

Why Attention Changes Everything

  • Eliminates the bottleneck — decoder accesses all encoder states
  • Provides interpretability — attention weights show where the model is "looking"
  • Creates shortcuts for gradient flow
  • Handles variable alignment between input and output
Key Innovation

Self-Attention — Attending to Yourself

Self-attention (Vaswani et al., 2017) is the core idea behind Transformers. Instead of attending to a separate encoder, each position in a sequence attends to all other positions in the same sequence.

Each input token is transformed into three vectors:

Query, Key, Value Projections Qᵢ = xᵢ · W_Q (what am I looking for?)
Kᵢ = xᵢ · W_K (what do I contain?)
Vᵢ = xᵢ · W_V (what do I provide?)
Scaled Dot-Product Self-Attention Attention(Q, K, V) = softmax(QKᵀ / √dₖ) · V

The QKV Analogy

Think of a library search:

  • Query Q = your search question
  • Key K = index cards on each book
  • Value V = the actual book content

You compare your query against all keys (dot product), normalize the scores (softmax), then retrieve a weighted combination of values.

Why √dₖ Scaling?

Without scaling, for large dₖ, the dot products grow large, pushing softmax into regions where gradients are extremely small. Dividing by √dₖ keeps the variance of the dot products at 1, maintaining healthy gradients.

Architecture Detail

Multi-Head Attention

A single attention head can only focus on one type of relationship. Multi-head attention runs h parallel attention heads, each with different learned projections, allowing the model to jointly attend to information from different representation subspaces:

Multi-Head Attention headi = Attention(QW_Qⁱ, KW_Kⁱ, VW_Vⁱ)

MultiHead(Q,K,V) = Concat(head₁,...,headₕ)W_O

Each head operates on a reduced dimension: if model dimension is d_model = 512 and h = 8 heads, each head works with dₖ = d_model/h = 64 dimensions.

What Different Heads Learn

Research shows that different heads specialize in different linguistic phenomena:

  • Head 1 might attend to syntactic dependencies (subject-verb)
  • Head 2 might track positional proximity (nearby words)
  • Head 3 might capture semantic similarity
  • Head 4 might handle coreference (pronouns to antecedents)

The final linear projection W_O learns to combine these diverse perspectives into a unified representation.

Dimensions W_Qⁱ, W_Kⁱ ∈ ℝ^(d_model × dₖ)
W_Vⁱ ∈ ℝ^(d_model × dᵥ)
W_O ∈ ℝ^(h·dᵥ × d_model)
V

The Transformer

"Attention Is All You Need" — Vaswani et al. (2017). The architecture that changed everything.

Architecture

The Transformer — No Recurrence At All

The Transformer's radical insight: discard recurrence entirely. Process the whole sequence at once using only attention and feedforward layers.

Encoder Block (×N)

x' = LayerNorm(x + MultiHeadAttn(x,x,x))
output = LayerNorm(x' + FFN(x'))

Decoder Block (×N)

x' = LayerNorm(x + MaskedMultiHeadAttn(x,x,x))
x'' = LayerNorm(x' + MultiHeadAttn(x',enc,enc))
output = LayerNorm(x'' + FFN(x''))

Why Transformers Win

  • Parallelization: No sequential dependency — all positions computed simultaneously. Training on GPUs is orders of magnitude faster.
  • Global context: Every position directly attends to every other position. No information loss over distance.
  • Constant path length: Signal travels from any position to any other in O(1) layers, vs O(n) for RNNs.
  • Scalability: Can be scaled to billions of parameters effectively.

Feed-Forward Network (FFN)

FFN(x) = max(0, xW₁ + b₁)W₂ + b₂
(Two linear layers with ReLU activation)

Typically d_ff = 4 × d_model (e.g., 2048 for d_model=512)

Essential Component

Positional Encoding

Since Transformers have no recurrence or convolution, they have no inherent notion of token order. Without positional information, "dog bites man" and "man bites dog" would be identical to the model.

The original Transformer uses sinusoidal positional encodings, added directly to the input embeddings:

Sinusoidal Positional Encoding PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

Each dimension i corresponds to a sinusoid with a different frequency, creating a unique "fingerprint" for each position.

Why Sinusoids?

  • Unique positions: Each position gets a distinct encoding
  • Relative positions: PE(pos+k) can be expressed as a linear function of PE(pos), letting the model easily learn relative distances
  • Extrapolation: Can generalize to sequence lengths not seen during training
  • Bounded: Values stay in [-1, 1] regardless of sequence length

Modern Alternatives

  • Learned embeddings — trainable position vectors (BERT, GPT)
  • RoPE — Rotary Position Embedding (LLaMA, used in most modern LLMs)
  • ALiBi — Attention with Linear Biases (BLOOM)
Training Stability

Residual Connections & Layer Normalization

Two critical components that make deep Transformers trainable:

Residual Connection (Skip Connection) output = x + SubLayer(x)

Residual connections create gradient highways — identical in spirit to why the LSTM cell state works. The gradient can flow directly through the addition, bypassing potentially problematic sublayers.

Layer Normalization LayerNorm(x) = γ ⊙ (x − μ) / (σ + ε) + β

where μ = mean(x), σ = std(x)
γ, β are learned scale and shift parameters

Layer norm stabilizes the distribution of activations, preventing internal covariate shift and enabling much deeper networks.

Pre-Norm vs Post-Norm

Post-Norm (original Transformer):

output = LayerNorm(x + SubLayer(x))

Pre-Norm (GPT-2, modern LLMs):

output = x + SubLayer(LayerNorm(x))

Pre-norm is now standard because it provides more stable training at scale — the residual path is never normalized, ensuring clean gradient flow.

The Pattern Repeats

Notice the recurring theme: from LSTM cell states to ResNet skip connections to Transformer residuals — the solution to training deep networks always involves additive shortcuts for gradient flow.

Evolution

From Transformers to GPT — The Decoder-Only Revolution

The original Transformer used both encoder and decoder stacks. Subsequent work explored three variants:

Encoder-Only (BERT, 2018)

Uses only the encoder stack with bidirectional self-attention. Trained via Masked Language Modeling (predict masked tokens). Excels at understanding tasks: classification, NER, QA.

Encoder-Decoder (T5, BART)

Full architecture. Encoder bidirectional, decoder causal. Best for sequence-to-sequence tasks: translation, summarization.

Decoder-Only (GPT, 2018)

Uses only the decoder with causal (masked) self-attention. Trained via next-token prediction. Emerged as the dominant paradigm for generative AI.

GPT's Causal Masking

In GPT, each token can only attend to tokens before and including itself — never to future tokens. This is enforced by masking future positions with −∞ before softmax:

Causal Attention Mask Maskᵢⱼ = { 0 if j ≤ i, −∞ if j > i }

Attention = softmax((QKᵀ + Mask) / √dₖ) · V

This makes GPT autoregressive — it can generate text one token at a time, each conditioned on all previous tokens.

GPT Training Objective

Causal Language Modeling L = −Σᵢ log P(xᵢ | x₁, x₂, ..., xᵢ₋₁; θ)

Maximize the probability of each token given all preceding tokens.

Synthesis

Key Mathematical Takeaways

The Gradient Problem

∂hₜ/∂hₖ = Πⱼ diag(tanh′) · Wₕₕ

Products of matrices with spectral radius ≠ 1 cause gradients to vanish or explode. This is the fundamental limitation of vanilla RNNs and the problem that LSTM/GRU, residual connections, and attention all solve in different ways.

The Gating Solution

Cₜ = fₜ ⊙ Cₜ₋₁ + iₜ ⊙ C̃ₜ

Additive cell state updates with learned gates create gradient highways. The gradient through the cell state is simply the product of forget gates — which can be trained to stay near 1 for long-term memory.

The Attention Solution

Attn = softmax(QKᵀ/√dₖ)V

Direct, content-based access to all positions eliminates sequential bottlenecks entirely. O(1) path length for any pair of positions. Combined with residual connections, enables training 100+ layer networks.

The thread connecting RNN → LSTM → GRU → Attention → Transformer → GPT is a single idea: create shorter, more direct paths for information and gradients to flow. Every major architecture innovation in sequence modeling can be understood as an engineering response to this principle.

Current State

The Modern Landscape — Beyond Vanilla Transformers

Efficient Attention Variants

Standard attention is O(n²) in sequence length. Modern approaches reduce this:

  • Flash Attention — hardware-aware exact attention (O(n²) but 2-4× faster via tiling)
  • Sparse Attention — attend only to subset of positions
  • Linear Attention — kernel approximations, O(n) complexity
  • Ring Attention — distribute long sequences across devices

State Space Models (SSMs)

Mamba and similar SSMs bring back the recurrent paradigm with O(n) complexity and no attention. They use structured state matrices to maintain long-range memory — a fascinating full-circle return to sequential processing, but with the mathematical elegance to avoid vanishing gradients.

Key Innovations in Modern LLMs

  • RoPE: Rotary Positional Embeddings — encode position through rotation in complex space
  • GQA: Grouped Query Attention — share KV heads to reduce memory
  • MoE: Mixture of Experts — activate only subset of parameters per token
  • SwiGLU: Improved activation function for FFN layers
  • RMSNorm: Simpler, faster normalization
  • KV Cache: Cache key-value pairs for efficient autoregressive generation

The Hybrid Future

The latest architectures (Jamba, Griffin, RWKV) combine Transformer layers with SSM layers, leveraging the strengths of both paradigms — attention for complex reasoning, recurrence for efficient long-context processing.
