Understanding Assumptions, Applications & Cost Functions

What is Linear Regression?

Linear regression is a fundamental statistical method used to model the relationship between a dependent variable (Y) and one or more independent variables (X).

Y = β₀ + β₁X₁ + β₂X₂ + ... + βₙXₙ + ε

Key Components:

  • Y: Dependent variable (what we're predicting)
  • β₀: Intercept (value of Y when all X = 0)
  • β₁, β₂, ..., βₙ: Coefficients (slopes)
  • X₁, X₂, ..., Xₙ: Independent variables (predictors)
  • ε: Error term (residual)

Types of Linear Regression

Simple Linear Regression

Y = β₀ + β₁X + ε

Models relationship between one independent variable and the dependent variable.

Example: Predicting house price based on square footage.

Multiple Linear Regression

Y = β₀ + β₁X₁ + β₂X₂ + ... + ε

Models relationship between multiple independent variables and the dependent variable.

Example: Predicting house price based on square footage, location, and age.

Key Assumptions of Linear Regression (1/2)

1. Linearity

The relationship between independent and dependent variables must be linear.

Check: Use scatter plots or residual plots. If the relationship curves, consider polynomial regression or transformations.

2. Independence

Observations must be independent of each other. No observation should influence another.

Violation Example: Time series data where today's value depends on yesterday's value (autocorrelation).

3. Homoscedasticity

The variance of residuals should be constant across all levels of independent variables.

Check: Plot residuals vs. fitted values. The spread should be uniform, not funnel-shaped.

Key Assumptions of Linear Regression (2/2)

4. Normality of Residuals

Residuals (errors) should follow a normal distribution with mean zero.

Check: Use Q-Q plots or Shapiro-Wilk test. For large samples, this assumption is less critical due to Central Limit Theorem.

5. No Multicollinearity

Independent variables should not be highly correlated with each other.

Check: Calculate Variance Inflation Factor (VIF). VIF > 10 indicates problematic multicollinearity.
Impact: Makes it difficult to determine the individual effect of each variable.

6. No Autocorrelation

Residuals should not be correlated with each other (particularly important for time series).

Check: Use Durbin-Watson test. Values around 2 indicate no autocorrelation.

1. Sales Forecasting

Problem: Predict future sales based on advertising spend, season, and economic indicators.

Sales = β₀ + β₁(TV_Ads) + β₂(Online_Ads) + β₃(Season) + ε

2. Stock Price Prediction

Problem: Estimate stock returns based on market indices, company fundamentals, and sector performance.

3. Risk Assessment

Problem: Determine credit risk scores based on income, debt ratio, credit history, and employment status.

4. Price Optimization

Problem: Find optimal pricing by analyzing relationship between price points and demand volume.

Example: Predicting diabetes risk or hospital readmission rates.

Cost Function: The Foundation

The cost function (also called loss function) measures how well our linear regression model fits the data. It quantifies the difference between predicted and actual values.

Why Do We Need a Cost Function?

  • To measure model performance objectively
  • To guide the optimization process (finding best β values)
  • To compare different models
  • To detect overfitting or underfitting
Goal: Minimize the cost function to find the best-fitting line that makes the most accurate predictions.

Mean Squared Error (MSE)

The most commonly used cost function for linear regression.

MSE = (1/n) Σ(yᵢ - ŷᵢ)²

Where:
• n = number of observations
• yᵢ = actual value
• ŷᵢ = predicted value
• (yᵢ - ŷᵢ) = residual/error

Key Characteristics:

  • Squares errors: Penalizes large errors more heavily (quadratic)
  • Always positive: Squaring eliminates negative values
  • Differentiable: Easy to optimize using gradient descent
  • Sensitive to outliers: Large errors get squared, having disproportionate impact

Alternative Cost Functions

1. Mean Absolute Error (MAE)

MAE = (1/n) Σ|yᵢ - ŷᵢ|

Advantage: Less sensitive to outliers than MSE

Disadvantage: Not differentiable at zero (harder to optimize)

2. Root Mean Squared Error (RMSE)

RMSE = √[(1/n) Σ(yᵢ - ŷᵢ)²]

Advantage: Same units as the dependent variable, easier to interpret

3. R² (Coefficient of Determination)

R² = 1 - (SS_res / SS_tot)

Interpretation: Proportion of variance explained by the model (0 to 1)

R² = 0.85 means 85% of variance is explained by the model

Optimization: Gradient Descent

Gradient descent is an iterative optimization algorithm used to minimize the cost function by finding the optimal coefficients (β values).

How It Works:

  1. Initialize coefficients with random values
  2. Calculate the cost function
  3. Compute the gradient (partial derivatives)
  4. Update coefficients in the direction that reduces cost
  5. Repeat until convergence
βⱼ = βⱼ - α * (∂J/∂βⱼ)

Where:
• α = learning rate (step size)
• ∂J/∂βⱼ = partial derivative of cost function
Learning Rate: Too large → overshooting; Too small → slow convergence

Evaluating Model Performance

1. Training vs. Testing Error

Always split data into training and testing sets to evaluate generalization.

Overfitting: Low training error, high testing error
Underfitting: High training error, high testing error

2. Adjusted R²

Penalizes adding unnecessary variables to the model.

3. Residual Analysis

  • Plot residuals vs. fitted values (check homoscedasticity)
  • Q-Q plot (check normality)
  • Residuals vs. time (check independence)

4. Cross-Validation

K-fold cross-validation provides robust performance estimates by testing on multiple data splits.

Practical Implementation Tips

Data Preparation

  • Feature Scaling: Normalize or standardize features for faster convergence
  • Handle Missing Data: Impute or remove carefully
  • Encode Categorical Variables: Use one-hot encoding or label encoding
  • Outlier Detection: Identify and handle outliers appropriately

Model Improvement

  • Feature Engineering: Create interaction terms or polynomial features
  • Regularization: Use Ridge (L2) or Lasso (L1) to prevent overfitting
  • Feature Selection: Remove irrelevant or redundant features
  • Transform Variables: Log, square root, or Box-Cox transformations

Summary

Key Takeaways:

  • Linear regression models linear relationships between variables
  • Six critical assumptions must be validated for reliable results
  • Applications span business, healthcare, science, and technology
  • MSE is the primary cost function, measuring prediction accuracy
  • Gradient descent optimizes coefficients by minimizing cost
  • Proper evaluation prevents overfitting and ensures generalization
