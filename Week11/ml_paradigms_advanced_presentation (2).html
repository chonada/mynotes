Supervised Learning: Technical Deep Dive

Mathematical Foundation

Learning Objective: Find function f: X → Y that minimizes empirical risk
Remp(f) = 1/n Σ L(f(xi), yi)

Classification

Output: Discrete classes

Loss Functions:

  • Cross-Entropy Loss
  • Hinge Loss (SVM)
  • Focal Loss (imbalanced)

Regression

Output: Continuous values

Loss Functions:

  • MSE (Mean Squared Error)
  • MAE (Mean Absolute Error)
  • Huber Loss (robust)

Core Algorithms

K-Means Clustering

Objective: Minimize within-cluster variance

Formula: J = Σ ||x - μk||²

Time: O(n·k·i·d)

DBSCAN

Parameters: ε (radius), MinPts

Advantage: Finds arbitrary shapes

Use: Spatial data, anomalies

PCA

Goal: Maximize variance preservation

Applications: Feature extraction, visualization

Autoencoders

Architecture: Encoder → Latent → Decoder

Types: Vanilla, VAE, Denoising

Semi-Supervised & Self-Supervised Learning

Semi-Supervised Learning Techniques

1. Pseudo-Labeling

Train on labeled → Predict unlabeled → Select high-confidence → Retrain

Threshold: 95%+ confidence

2. Co-Training

Two models on different feature views

Each labels data for the other

3. Consistency Regularization

Consistent predictions for perturbed inputs

Loss: L = Lsup + λLcons

Active, Transfer & Few-Shot Learning

Active Learning: Strategic Data Acquisition

Uncertainty Sampling

Least Confidence: 1 - max(P(y|x))

Margin: P(y₁|x) - P(y₂|x)

Entropy: -Σ P(y|x) log P(y|x)

Query-by-Committee

Train ensemble of models

Select samples with highest disagreement

Transfer Learning Pipeline

Pre-train on Source

Learn from large datasets (ImageNet: 14M images)

Transfer Knowledge

Adapt learned features to new domain

Fine-tune on Target

Optimize for specific task with minimal data

Complete ML Development Cycle

Problem Definition

Define: Business objective, success criteria, constraints

Example: Reduce churn by 15% → Predict churn in 30 days

Data Collection

Sources: Internal DBs, APIs, web scraping, open datasets

Quality: Completeness, validity, accuracy, consistency

Key: Handle missing values, outliers, scaling

Model Selection

Choose: Algorithm based on problem type, data size

Start: Simple (LogReg, RF) → Complex (Deep Learning)

Split: Train (70%), Validation (15%), Test (15%)

Validate: K-fold cross-validation

Evaluation & Tuning

Metrics: Accuracy, F1, ROC-AUC for classification

Tune: Grid search, random search, Bayesian optimization

Deployment & Monitoring

Deploy: REST API, batch, edge deployment

Monitor: Performance, data drift, system metrics

Training Strategy

from sklearn.model_selection import StratifiedKFold, cross_val_score from sklearn.ensemble import RandomForestClassifier # Stratified K-Fold Cross-Validation cv = StratifiedKFold(n_splits=5, shuffle=True) # Train model clf = RandomForestClassifier(n_estimators=200) # Evaluate with multiple metrics metrics = ['accuracy', 'precision', 'recall', 'f1'] for metric in metrics: scores = cross_val_score(clf, X_train, y_train, cv=cv, scoring=metric) print(f"{metric}: {scores.mean():.4f}")

Overfitting vs Underfitting

⚠️ Underfitting

  • Low training & test accuracy
  • Model too simple
  • Fix: Increase complexity

⚠️ Overfitting

  • High train, low test accuracy
  • Memorizes training data
  • Fix: Regularization, more data

Model Evaluation & Hyperparameter Tuning

Classification Metrics

Metric Formula When to Use
Accuracy (TP + TN) / Total Balanced datasets only
Precision TP / (TP + FP) Cost of FP high (spam)
Recall TP / (TP + FN) Cost of FN high (cancer)
F1-Score 2×(P×R)/(P+R) Imbalanced datasets
ROC-AUC Area under ROC Probability calibration

Hyperparameter Tuning with Optuna

import optuna def objective(trial): params = { 'n_estimators': trial.suggest_int('n_estimators', 100, 500), 'max_depth': trial.suggest_int('max_depth', 5, 30), 'min_samples_split': trial.suggest_int('min_samples_split', 2, 20) } clf = RandomForestClassifier(**params) score = cross_val_score(clf, X_train, y_train, cv=5) return score.mean() study = optuna.create_study(direction='maximize') study.optimize(objective, n_trials=100)
F1: 0.64 → 0.82

Development Best Practices

Establish Baseline

Start with simple models before complex deep learning

Experiment Tracking

Common Pitfalls to Avoid

❌ Don't

  • Overfit to test set
  • Data leakage
  • Ignore class imbalance
  • No versioning
  • Complex models first

✅ Do

  • Proper validation
  • Time-aware splits
  • Appropriate metrics
  • Version everything
  • Start simple
